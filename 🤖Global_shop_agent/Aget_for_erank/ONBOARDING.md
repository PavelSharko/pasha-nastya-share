# Инструкция Онбординга Агента eRank 🚀

Этот документ описывает процесс инициализации и работы агента для сбора данных с eRank.

## 1. Процесс инициализации (Onboarding)
При первом запуске или потере сессии агент ОБЯЗАН:
1.  **Спросить пользователя**, какой браузер использовать для поиска кукисов (по умолчанию: `Google Chrome` или `Kometa`).
2.  **Попытаться найти файл сессии/кукисов** в системных директориях выбранного браузера.
3.  **Проверить валидность сессии**: перейти на `https://members.erank.com/dashboard`.
    *   Если видна страница логина -> Сообщить пользователю: "Сессия не найдена. Пожалуйста, зайдите в [Название браузера], авторизуйтесь на erank.com и перезапустите меня".
    *   Если виден дашборд -> Установить флаг `status: authorized`.

## 2. Алгоритм работы (Extraction Workflow)
1.  **Запуск браузера**: Использовать Playwright в режиме `persistent_context` (с существующим профилем браузера) или загружать куки из JSON.
2.  **Таймауты**: 
    *   Ожидание загрузки страницы: 30 секунд.
    *   Ожидание селектора (таблицы данных): 15 секунд.
    *   Если таймаут вышел -> Сделать скриншот ошибки и спросить пользователя, что делать дальше.
3.  **Сбор данных**: 
    *   Основная цель: забрать содержимое страницы (HTML или JSON из сетевых запросов).
    *   Сохранение: записывать результат в файл `raw_data.html` или `data.json` в папку агента.

## 3. Флаги состояния (State Management)
Файл `agent_state.json` (создается агентом):
- `last_browser`: Название выбранного браузера.
- `auth_status`: `true/false`.
- `last_sync`: Дата последней успешной выгрузки.

---
*Инструкция для разработчика: При падении скрипта из-за отсутствия кукиса — не висеть в процессе, а выводить четкую ошибку в консоль/чат.*
