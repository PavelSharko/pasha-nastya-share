# 🗺️ Roadmap: eRank Agent v2.0

Список задач по улучшению агента для глубокого анализа тегов.

## 🚀 Основной функционал (Updated 08.06.2026)
- [ ] **Гибридный запуск (Входные данные)**:
    - Если запущен человеком (без флага `--auto`): запрашивать тег через `input()` в консоли.
    - Если запущен агентом/системой: брать входные данные (очередь тегов) напрямую из базы данных.
- [ ] **Полные отчеты**: Снимать ограничение в 50-100 строк в MD, сохранять все доступные теги (до 500-1000+).
- [ ] **Запись в БД (Интеграция)**:
    - Обучить агента не только делать MD-отчет, но и правильно структурировать и записывать результаты анализа в базу данных (таблицы тегов/аналитики).
- [ ] **Локализация**: Добавить автоматический перевод тегов на русский язык в финальный отчет и базу данных.

## 📊 Аналитика и Интеллект
- [ ] **Инструкция как Хелп**: Переписать `Промт для тегов.md` в формате системной инструкции для агента (Help Guide), чтобы он понимал разницу между "Массовым спросом" и "Узкой нишей".
- [ ] **Экспертный отбор (Strategy Logic)**:
    - Реализовать алгоритм фильтрации на основе цветов и показателей согласно стратегиям.
    - Отбор Топ-15 лучших тегов с обоснованием.
    - Исключение "шумных" запросов и приоритет длинным фразам (long-tail keywords).

## 🛠 Техническое и Стабильность
- [ ] **Авто-авторизация**: Улучшить поиск кукисов в разных браузерах (Chrome/Kometa/Safari).
- [ ] **Мульти-региональность**: Возможность выбрать сразу несколько стран и получить сводный отчет.
- [ ] **Графики спроса**: Собирать данные по месяцам (Monthly Search Trend), чтобы видеть сезонность.
- [ ] **Умные тайм-ауты**: Улучшить логику ожидания для очень медленных страниц eRank.
