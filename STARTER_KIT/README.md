# 🚀 Инструкция по установке Hybrid LLM Wiki Starter Kit

Добро пожаловать в Starter Kit! Этот набор поможет вам быстро развернуть систему "Второго Мозга" с продвинутыми навыками автоматизации.

## 📦 Что внутри
- `GEMINI.md`: Главные правила и протоколы работы агента.
- `.agents/skills/`: Набор мощных инструментов (управление сессиями, создание новых навыков и др.).
- `INIT_PROMPT.md`: Инструкция для первой инициализации агента.
- Базовая структура папок (`RAW`, `TODO`, шаблоны доменов).

## 🛠 Как установить (Инструкция для Пользователя)
1. Скопируйте содержимое папки `STARTER_KIT` в корень вашего рабочего пространства (Obsidian Vault или просто рабочая папка).
2. Запустите ваш Gemini CLI в этой директории.
3. Отправьте агенту следующий промпт:
   > "Прочитай файл `INIT_PROMPT.md` и `README.md`. Создай текстовый файл `INSTALLATION_PLAN.md`, распиши в нем пошаговый план внедрения этого стартера в мою текущую среду и начни выполнять его, отмечая прогресс галочками."

---

## 🤖 Инструкция для Агента (Critical Rules)

При внедрении этого стартера ты ОБЯЗАН соблюдать следующие принципы:

### 1. Сохранение и Обогащение
- **НЕ УДАЛЯЙ** существующие инструкции пользователя (`GEMINI.md` или другие файлы настроек), если они уже есть. Твоя задача — **обогатить** их полезными инструментами и протоколами из этого стартера.
- Если в текущей среде уже есть файл `GEMINI.md`, проведи сравнение. Если подходы из стартера противоречат текущим настройкам пользователя — **ОСТАНОВИСЬ и СПРОСИ**, какой подход оставить. Если конфликтов нет, просто объедини лучшее из двух миров.

### 2. Уважение к существующей структуре
- Если у пользователя **уже есть** свои папки и ниши (домены), не пытайся их заменить на шаблонные `DOMAIN_1_*`. Вместо этого:
  - Изучи существующие папки.
  - Впиши их в новую систему (обнови пути в `GEMINI.md`).
  - Предложи, как можно улучшить текущую структуру, используя принципы из `GEMINI.md`.

### 3. План внедрения (Installation Plan)
- **ОБЯЗАТЕЛЬНО** перед началом любых изменений создай файл `INSTALLATION_PLAN.md`.
- В плане опиши:
  - Какие файлы ты собираешься объединить.
  - Какие скиллы подключаешь.
  - Как ты адаптируешь структуру под текущие папки пользователя.
- Иди строго по пунктам, помечая выполненные шаги `[x]`. Это обеспечит прозрачность для владельца.

### 4. Конфликты и подтверждения
- Любое действие, которое может привести к удалению или значительному изменению существующей логики пользователя, должно быть подтверждено. 
- Твоя цель: сделать процесс установки бесшовным ("Seamless"), чтобы пользователь почувствовал прирост возможностей, а не потерю старых настроек.
