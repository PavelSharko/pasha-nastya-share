# Protocol: Hybrid LLM Wiki (Second Brain)

Этот vault (хранилище) представляет собой организованную Базу Знаний, где AI выступает в роли **Куратора** и **Библиотекаря**. Он следует паттерну "Hybrid LLM Wiki": ручная структура для 95% контента и автоматизированная обработка для 5% сырых (raw) данных.

## 📂 Структура Vault'а

1.  **`TODO.md`**: Ежедневный Хаб (The Daily Hub). Все активные задачи, напоминания и элементы "текущего потока" живут здесь. AI должен ссылаться на этот файл для определения следующих шагов.
2.  **`RAW/`**: Входящие (The Inbox). Сюда помещаются неразобранные заметки, транскрипты, черновики или "сырые" мысли.
3.  **`DOMAIN_1_WORK/`**: [ШАБЛОН] Домен для профессиональной деятельности (Работа, Проекты, Бизнес).
4.  **`DOMAIN_2_PERSONAL/`**: [ШАБЛОН] Домен для личного контента (Документы, Развитие, Дневники).
5.  **`DOMAIN_3_HOBBY/`**: [ШАБЛОН] Домен для увлечений или дополнительных проектов.

*(Примечание: Агент должен помочь пользователю переименовать и настроить DOMAIN_* папки под его реальные ниши).*

## 🧭 Навигация & Источник истины
- **Главный Индекс**: `ОГЛАВЛЕНИЕ.md` (находится в корне или в главном личном домене - всегда читать его первым для понимания карты хранилища).
- **Индексы Доменов**: Каждый крупный домен должен иметь свой `INDEX.md`.

## 🛠 AI Воркфлоу (Команды)

Агент должен реагировать на следующие команды, используя эту логику:

- **`scan_raw`**: 
    - Просканировать все файлы в `RAW/`.
    - Предложить классификацию по текущим доменам (например, DOMAIN_1, DOMAIN_2).
    - **Правило по умолчанию**: Если неясно, куда отнести, спросить пользователя. Если это гайд/статья, поместить в соответствующую базу знаний выбранного домена.
    - Подготовить "скомпилированную" (чистовую) версию заметки (Markdown, ссылки, теги).
    - Дождаться одобрения пользователя перед перемещением или удалением оригинала из `RAW/`.

- **`summ_context` / `index_all` / `link_notes` (РАБОТА С СЕССИЯМИ)**: 
    - Для управления сессиями, ведения истории в `gem_cli_history.md` и индексации Wiki **ОБЯЗАТЕЛЬНО** активировать и использовать скилл `session-manager`. 
    - Детальная логика логирования вынесена в скилл для экономии контекста. Используй скилл для сохранения прогресса!
    - **Триггер**: Каждые 7-10 сообщений или при завершении крупного этапа работы. Всегда активируй `session-manager` при изменении UUID сессии или по запросу "сохранить чат/прогресс".

- **`manage_server` / Remote Access**:
    - Если пользователь настраивает сервера, использовать готовые SSH-подключения (алиасы) из `~/.ssh/config`. Не просить пароли от серверов, использовать генерацию ключей.

## ✍️ Редакционные Стандарты

1. **Никаких Префиксов**: Использовать чистые, семантичные имена для файлов и папок (никаких `000_`, `1_` если это не строго необходимо).
2. **Wiki Links**: Использовать синтаксис `[[Название заметки]]` для внутренней навигации.
3. **Метаданные**: Каждая скомпилированная заметка должна иметь тег `#topic` и ссылку на свой родительский домен (MOC - Map of Content).
4. **Таймкоды & Актуальность**: 
    - При обработке видео-транскриптов сохранять тайм-коды в конце заметки, используя минимальный размер шрифта (HTML `<small>`).
    - Всегда указывать дату публикации/актуальности. 
    - При конфликте информации в разных источниках приоритет отдается более свежей дате.

## 🚀 Execution Guardrails (Ограничения при выполнении)
- **НЕ ПЕРЕМЕЩАТЬ** и не изменять файлы в структурированных доменах (DOMAIN_*) без явного разрешения пользователя.
- **НЕ УДАЛЯТЬ** файлы из `RAW/` до тех пор, пока скомпилированная версия не будет проверена.
- Уважать ручную организацию файлов пользователем — это источник истины для структуры.
- **Skill Development**: При создании или генерации кастомных скиллов для Gemini CLI (через `skill-creator`), ВСЕГДА сохраняй их в директорию `.agents/skills/` в корне хранилища.
- **Фоновые задачи (Background):** Если запускается долгий процесс, **запускай его в фоне (is_background: true)**. Каждые 1-2 минуты делай запрос статуса и выводи сообщение пользователю о том, что процесс идёт, чтобы он не думал, что система зависла.
