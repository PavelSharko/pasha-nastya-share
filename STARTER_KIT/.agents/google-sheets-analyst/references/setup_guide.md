# Настройка Google Sheets API (Одноразовая подготовка)

Чтобы ИИ-агент (Gemini CLI) мог читать ваши Google Таблицы напрямую, нужно создать "сервисный аккаунт" (виртуального пользователя-бота). Это делается один раз.

## Шаг 1: Google Cloud Console
1. Перейдите в [Google Cloud Console](https://console.cloud.google.com/).
2. Создайте новый проект (Create Project) и назовите его, например, `gemini-sheets-bot`.
3. В меню слева перейдите в **APIs & Services** -> **Library**.
4. Найдите и включите (Enable) два API:
   - `Google Sheets API`
   - `Google Drive API`

## Шаг 2: Создание ключа доступа (Сервисный аккаунт)
1. В меню слева перейдите в **APIs & Services** -> **Credentials**.
2. Нажмите **Create Credentials** -> **Service Account**.
3. Заполните имя (например, `gemini-bot`) и нажмите **Create and Continue**, затем **Done**.
4. Вы увидите созданный аккаунт в списке Service Accounts. Кликните по его Email.
5. Перейдите во вкладку **Keys** -> **Add Key** -> **Create new key**.
6. Выберите формат **JSON** и нажмите **Create**. Файл скачается на ваш компьютер.
7. Переименуйте этот файл в `credentials.json`.

## Шаг 3: Безопасное сохранение ключа
1. Поместите файл `credentials.json` в папку скилла, рядом со скриптом `read_sheet.py`. 
2. **ВАЖНО**: Убедитесь, что этот файл добавлен в `.gitignore`, если вы используете Git. Ключ нельзя публиковать в интернете!

## Шаг 4: Выдача прав боту на ваши таблицы
1. Откройте файл `credentials.json` как текст и найдите там строчку `client_email` (выглядит как `gemini-bot@gemini-sheets-bot...iam.gserviceaccount.com`).
2. Скопируйте этот email.
3. Откройте вашу Google Таблицу в браузере, нажмите зеленую кнопку **Поделиться** (Share).
4. Вставьте этот email и выдайте права **Читателя** (Viewer).

Всё! Теперь бот имеет доступ конкретно к этой таблице и сможет отвечать на ваши запросы в терминале.
