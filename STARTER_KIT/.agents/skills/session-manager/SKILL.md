# Skill: Session Manager & Librarian

Этот скилл превращает Gemini в дисциплинированного библиотекаря, который следит за тем, чтобы ни одна сессия не потерялась, а база знаний (Wiki) оставалась связанной.

## 🛠 Основные команды (Триггеры)

### 1. `summ_context` / Сохранение прогресса
Выполняй этот протокол каждые 7-10 сообщений для активных сессий или чаще для новых.

**Протокол выполнения:**
1. **Sync Title**: Вызови `update_topic(title="Имя сессии")` для синхронизации заголовка.
2. **UUID Discovery**: Запусти `gemini --list-sessions`. Найди текущий UUID, сопоставив заголовок сессии.
3. **History Logging (`gem_cli_history.md`)**:
    - **CRITICAL**: Никогда не удаляй старые записи.
    - Если сессия уже есть в файле: Добавь новые достижения в поле **Суть** через запятую или с новой строки. Обнови **Команда возврата** актуальным UUID.
    - Если UUID изменился (старая сессия недоступна): Создай **новый блок** в истории, сохранив описание предыдущих этапов. Добавь пометку "(UUID обновлен)".
    - Если сессия новая: Создай новый блок в начале списка (после заголовка файла).
4. **Adaptive Hook**: 
    - Если сессия еще НЕ залогирована или пользователь не подтвердил `/chat save`: выводи "The Hook" в конце каждого ответа.
    - Если сессия уже залогирована: выводи "The Hook" только раз в 7-10 сообщений.

**The Hook (Шаблон):**
> 💾 **Сохраним прогресс, чтобы ты не потерял этот чат?**
> 1. Вставь в терминал: `/chat save "имя-сессии"`
> 2. А затем напиши мне: `я выполнил /chat save "имя-сессии"`

---

### 2. `index_all` / Индексация Wiki
Используй для поддержания связности базы знаний.

**Протокол выполнения:**
1. Просканируй новые файлы в `АЙТИ/`, `ТРЕНЕР/`, `ЛИЧНОЕ/`.
2. Обнови центральный индекс `ЛИЧНОЕ/ОГЛАВЛЕНИЕ.md` и отраслевой индекс `ТРЕНЕР/INDEX.md`.
3. Убедись, что новые заметки имеют обратные ссылки на родительские MOC (Map of Content).
4. Проверь "здоровье" связей (нет ли битых ссылок `[[...]]`).

---

### 3. `link_notes` / Умное связывание
Анализируй контекст текущей задачи и предлагай связи с существующими заметками (например, "Это пересекается с методикой из [[Методология_Мета_Контекста]]").

## ✍️ Редакционные стандарты
- Использовать Wiki-ссылки: `[[Имя заметки]]`.
- Основной текст: Русский. Технические термины: English.
- Метаданные: Каждая скомпилированная заметка должна иметь тег `#topic` и ссылку на родительский домен.
